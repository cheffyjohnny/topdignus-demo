# 견적서 자동화 기능 기획

작성일: 2026-04-17

---

## 개요

Gmail(Google Workspace)로 수신된 발주 요청 메일을 읽어 견적서 엑셀 파일을 반자동으로 생성하는 기능.
Internal dashboard(`/dashboard`)에 통합.

---

## 핵심 기능

### 1. 미읽은 메일 반자동 견적서 작성
- Gmail API로 미읽은 메일 감지
- 메일 타입 자동 분류 후 데이터 파싱
- 파싱 결과를 대시보드에 표시 → 사람이 검토 → 엑셀 생성

### 2. 수동 실행 트리거
- 대시보드에서 "미처리 발주 불러오기" 버튼으로 수동 실행
- 자동 주기 실행 + 수동 실행 병행

---

## 기존 엑셀 템플릿 구조

파일: `견적서-이주헌 개발/배관 ^ 필립산업/번호 ^^ 26수발거인(원본) ...xlsx`

| 시트 | 용도 | 자동화 우선순위 |
|------|------|----------------|
| `1.발주접수` | 발주의뢰처, 현장명, 납품장소, 인수자 등 | 높음 |
| `3.견적서` | 정식 견적서 (고객용) | 높음 |
| `4.거래명세서` | 납품 명세서 (고객용) | 높음 |
| `5.발주서` | 공급사(필립산업) 발주서 | 중간 |
| `2.고정구단가산출` | 차열재·실란트 계산 | 낮음 (추가 DB 필요) |
| `2-1.고정구단가산출<출력용>` | 위 계산 출력용 | 낮음 |
| `단가표` | 마스터 가격표 (365행) | 참조용 유지 |
| `ECOUNT(주문/발주입력)` | ERP 입력용 | 낮음 (코드 추가 필요) |

## 메일 파싱 아키텍처

```
Gmail API → 미읽은 메일 수신
  ↓
메일 타입 자동 분류
  ├── 첨부 xlsx  → xlsx 라이브러리 파싱 (코드)
  ├── 첨부 PDF   → pdf-parse (코드, 정확도 중간)
  ├── 텍스트     → 키워드/정규식 추출 (코드)
  └── 이미지     → Google Drive OCR (무료, Google Workspace 내장)
          └── Google Drive API 임시 업로드
              → Google Docs 변환 (Drive 내장 OCR, 한국어 지원)
              → 텍스트 추출 → 키워드 파싱
  ↓
대시보드 "파싱 결과 검토" 화면 표시
  → 사람이 데이터 확인/수정
  → 엑셀 생성 버튼 클릭
  → xlsx 다운로드
```

### OCR 전략
- **Claude API 미사용** (추가 비용 없이 구현)
- Google Workspace 내장 OCR 활용 (무료)
- 대상: 인쇄/디지털 출력 발주서 이미지 (손글씨 제외)
- 인쇄물 한국어 OCR 정확도: 85~90% 수준
- 나머지 오류는 사람이 검토 단계에서 수정

### 이미지 발주서 파싱 가능 필드 (예시)
```
수신, 회사명/담당자, 납기일자, 프로젝트명, 납품주소,
품목명, 규격(배관×슬리브), 수량
```

## DB 추가 작업

### 현재 DB로 커버되는 항목
- `quotes`: 발주의뢰처, 현장명, 납품지, 수신자, 납기일, 총금액
- `quote_items`: 품목명, 규격, 단위, 수량, 단가, 금액

### 추가가 필요한 항목
| 항목 | 용도 | 방법 |

- Google Cloud Console 프로젝트 생성
